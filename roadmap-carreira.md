Ótima ideia! Colocar o **GitHub** logo no início vai acelerar sua visibilidade e já te dar prática com versionamento e publicação de projetos. Aqui está o roadmap ajustado para **120 dias**, começando pelo GitHub:

---

📅 **ROADMAP DE 120 DIAS (GitHub em primeiro lugar)**  

ADAPTADO PARA: 8 horas/semana  

**MÊS 1 - GITHUB E FUNDAMENTOS**  

SEMANA 1-2:  
- Criar conta no GitHub  
- Aprender comandos básicos de Git (clone, commit, push, pull)  
- Criar seu primeiro repositório e publicar um projeto simples (HTML + CSS)  

SEMANA 3-4:  
- Revisar HTML básico  
- Criar páginas simples com CSS e subir no GitHub Pages  

**MÊS 2 - JAVASCRIPT E PRÁTICA**  

SEMANA 5-6:  
- Introdução ao JavaScript  
- Exercícios práticos de manipulação do DOM e versionamento no GitHub  

SEMANA 7-8:  
- Criar pequenos projetos (formulários, calculadora)  
- Documentar cada projeto com README.md no GitHub  

**MÊS 3 - FRAMEWORKS E PROJETOS**  

SEMANA 9-10:  
- Introdução ao React  
- Construir componentes reutilizáveis e publicar no GitHub  

SEMANA 11-12:  
- Criar projeto React integrado com API  
- Versionar e documentar no GitHub com issues e branches  

**MÊS 4 - PORTFÓLIO E PREPARAÇÃO**  

SEMANA 13-14:  
- Desenvolver projeto completo (To-Do App ou Dashboard)  
- Publicar no GitHub e Vercel com README detalhado  

SEMANA 15-16:  
- Preparar currículo e LinkedIn com link para GitHub  
- Simular entrevistas técnicas e revisar fundamentos  

---

🚀 **PROJETO DE PORTFÓLIO**

PROJETO: **Aplicação de Lista de Tarefas (To-Do App)**  

O QUE FAZER:  
Criar uma aplicação web que permita adicionar, editar e remover tarefas, com persistência local.  

ENTREGÁVEIS:  
- Interface responsiva  
- CRUD de tarefas  
- Deploy online no GitHub Pages ou Vercel  

CRITÉRIOS DE ACEITAÇÃO:  
- Funcionalidade completa sem erros  
- Código organizado e comentado  
- Publicação acessível via link do GitHub  

DICA: Use README.md bem estruturado para explicar o projeto e impressionar recrutadores.
